# PROJECT BLUEPRINT - stackyrd-nano-NANO

## 1. PROJECT OVERVIEW

**stackyrd-nano-nano** is a minimal, compact Go boilerplate application with clean architecture. Stripped down version of original stackyrd-nano, keeping only core foundations while maintaining the original folder structure. Blank slate ready for custom feature addition.

### Key Characteristics:
- **Language**: Go (Golang) 1.25+
- **Framework**: Gin for HTTP services
- **Architecture**: Clean, layered, modular
- **Size**: Ultra compact, minimal dependencies
- **Purpose**: Blank boilerplate for rapid application development
- **Status**: Ready to build & extend

## 2. ARCHITECTURE & DESIGN

### 2.1 System Architecture

```
Client → Handler → Request Binding → Validation → Business Logic → Response
```

### 2.2 Package Organization

All original folders preserved, contents simplified:

```
stackyrd-nano-nano/
├── cmd/app/           - Application entry point
│   ├── main.go        - Simplified entry point
│   ├── constants.go   - Centralized constants
│   ├── config_manager.go - Configuration loading
│   └── application.go - Application lifecycle
├── config/            - Configuration management
├── internal/          - Private core logic
│   ├── middleware/    - HTTP middleware (basic only)
│   ├── server/        - HTTP server setup
│   └── services/      - Business services (empty, ready for your code)
│       └── modules/   - Keep directory, add your services here
└── pkg/               - Public reusable packages
    ├── logger/        - Logging utilities
    ├── request/       - Request binding & validation
    ├── response/      - Standard API responses
    └── utils/         - Core utility functions
```

### 2.3 Core Features Kept

Standardized Response Format
Request Validation System
Clean Configuration Management
Step-by-Step Application Execution
Proper Error Handling
Clean Code Architecture Principles

REMOVED FEATURES:
- All infrastructure integrations (Postgres, Mongo, Redis, Kafka, Minio, Grafana, Cron)
- Advanced middleware (JWT, Rate Limiting, Audit, Security Headers)
- TUI System & Live Dashboard
- Service Registry & Dependency Injection System
- Async Infrastructure System
- Monitoring, Metrics, Resilience Patterns
- Deployment scripts, Docker, Kubernetes, CI/CD
- All pre-built services & modules

## 3. CORE SYSTEMS

### 3.1 Standardized API Responses

All API responses follow consistent structure:
```json
{
  "success": true/false,
  "message": "Optional message",
  "data": {},
  "error": {
    "code": "ERROR_CODE",
    "message": "Error message"
  },
  "timestamp": 1672531200
}
```

### 3.2 Application Execution Flow

```
1. Configuration Loading
2. Banner Display
3. Port Validation
4. Logger Initialization
5. Server Startup
6. Serve requests
```

### 3.3 Server Implementation

- Clean Gin HTTP Server
- Custom 404 / 405 error handlers
- Recovery middleware
- Standard error responses
- Graceful shutdown support

## 4. GETTING STARTED

### 4.1 Setup

```bash
# Install dependencies
go mod tidy

# Run application
go run cmd/app/main.go
```

### 4.2 Adding Your First Service

1. Create service file in `internal/services/modules/`
2. Implement your handlers
3. Register routes in `internal/server/server.go`
4. Add any required configuration in `config.yaml`

### 4.3 Project Goals

stackyrd-nano-nano provides:
Clean folder structure
Standard API patterns
Proper error handling
Configuration system
Validation helpers
Logging system

**You add everything else according to your needs.**

## 7. CONCLUSION

stackyrd-nano-nano is the minimal foundation version of stackyrd-nano. All the bloat, infrastructure, and pre-built features have been stripped away leaving only the clean architecture core. Perfect starting point when you don't need all the enterprise features of original stackyrd-nano and just want a solid blank boilerplate to build upon.
